# Sprint 01 — Product Reset и фиксация scope

## Статус

- `Sprint 01` зафиксирован документально.
- Роли, контуры продукта и границы MVP/non-MVP формально определены.
- Этот документ является опорой для `Sprint 02`.

## Цель Sprint 01

Зафиксировать продуктовые границы, чтобы все следующие изменения в коде делались не "по ощущениям", а в рамках согласованного объёма.

## Зафиксированные роли

- `employee`: работает только со своими данными (журнал, планы, отчёты).
- `manager`: видит данные команды в режиме чтения и выгружает отчёты.
- `developer`: имеет технический контур и сопровождение платформы.

## Зафиксированные продуктовые контуры

- `day ops`: журнал и ежедневная операционная активность.
- `night works`: подготовка и сопровождение ночных изменений.
- `reports`: генерация отчётов по периодам и результатам работ.
- `templates`: библиотека типовых шаблонов.
- `team`: командная видимость для руководителя.
- `developer dashboard`: технический контур и метрики.

## MVP (обязательно в первой рабочей версии)

- Локальная auth с сессиями для development.
- Роли `employee/manager/developer` и базовые RBAC-ограничения.
- CRUD для `ActivityEntry` (журнал).
- CRUD для `NightWorkPlan/NightWorkBlock/NightWorkStep`.
- Базовые `PlanTemplate` и создание плана из шаблона.
- Генерация отчётов: `daily`, `weekly`, `custom range`.
- Экспорт минимум в `MD` и `TXT`.
- Архив с базовым поиском и фильтрацией.
- Developer-экран с базовыми health/метриками (MVP-уровень).

## Non-MVP (после первой рабочей версии)

- Полноценный production LDAP-режим с групповым маппингом.
- Экспорт `DOCX/PDF/XLSX` production-качества.
- Расширенная observability-платформа и error analytics.
- Расширенная настройка шаблонов форматирования документов.
- Продвинутые backup/restore сценарии и release-automation.

## Ограничения разработки с Sprint 01

- Любая новая задача должна ссылаться на `MVP` или `Non-MVP`.
- Задачи вне `MVP` не должны блокировать delivery текущих sprint-целей.
- Нельзя расширять scope задачи без явной фиксации в документации.

## Критерии готовности Sprint 01 (Definition of Done)

- README отражает актуальный scope и ссылки на правила.
- Есть отдельный документ с MVP/non-MVP границами.
- Роли и контуры не противоречат roadmap и product requirements.
- Команда может начать Sprint 02 без дополнительных уточнений по границам продукта.
